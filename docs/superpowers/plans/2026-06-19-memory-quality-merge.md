# 对话便签提取质量与去重合并实现计划

日期：2026-06-19

## 约束

- 严格按测试先行的红—绿—重构循环实施。
- 不读取或复制线上用户正文，不在测试中使用真实数据。
- 不修改 AstrBot 核心，不引入向量服务或新管理端口。
- 保持插件错误不影响正常聊天。

## 任务

1. 为 v2 到 v3 升级和变更历史写失败测试，实现迁移与事务型更新/合并接口。
2. 为本地候选召回写失败测试，实现同用户范围内的确定性候选选择。
3. 为结构化 `create/update/merge/noop` 解析和越权拒绝写失败测试，实现模型输出契约。
4. 为自动整理中的更新、合并、回滚和重试写失败测试，将新决策流程接入现有调度器。
5. 为质量整理预览、指纹过期保护、确认应用和历史读取写失败测试，实现后端 API。
6. 为页面契约与交互写失败测试，实现预览确认和历史视图。
7. 更新配置说明、README、CHANGELOG 和插件版本。
8. 运行完整测试、Ruff 与编译检查；检查 AstrBot 4.25.5 契约、插件加载/重载和持久化。

## 验收标准

- 近义便签可合并且保留最早 ID。
- 明确冲突可更新原便签并留下历史。
- 非候选、跨用户和非法操作不能写库。
- 预览后的并发变化会阻止应用。
- v2 数据升级后内容与 ID 不变。
- 所有既有与新增自动化检查通过，输出无警告或 traceback。
